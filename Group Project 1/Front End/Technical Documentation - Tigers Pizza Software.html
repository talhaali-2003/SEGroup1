Pizza Of Service - Technical Documentation

Overview

The Pizza Ordering and Inventory Management System is a Python-based application developed using the Tkinter library for graphical user interface (GUI) development. It provides functionalities for customers to order pizzas and for administrators to manage the inventory of pizza ingredients, track orders, and manage employee timesheets.

Features

  1. Pizza Ordering
    • Customers can select from a variety of pizzas, including Margherita, Pepperoni, Vegetarian, and Hawaiian.
    • They can customize their orders by choosing the size, crust type, and toppings.
    • The system validates customer inputs and checks inventory availability before placing an order.
    • Upon successful order placement, the system deducts ingredients from the inventory and saves the order details, including the customer's name and phone number, in a JSON file.
  2. Inventory Management
    • Administrators can view and manage the inventory of pizza ingredients.
    • They can see the current stock levels of each ingredient.
    • The system calculates and displays the total revenue earned from all orders.
  3. Employee Timesheet Management
    • Administrators can view and manage employee timesheets.
    • They can assign employees to different timeslots throughout the day.
    • Timesheets are organized into four timeslots: 6 AM - 10 AM, 10 AM - 2 PM, 2 PM - 6 PM, and 6 PM - 10 PM.

Implementation Details

Technologies Used

  • Python: The core programming language used for developing the application logic.
  • Tkinter: The Python library used for creating the graphical user interface.
  • JSON: Used for storing order data persistently in a file.
  • Datetime: Utilized for timestamping orders with the current date and time.

Classes and Modules

  • PizzaOrderingSystem Class: Manages the pizza ordering functionality, including GUI creation, order placement, and inventory deduction.
  • InventoryManagementSystem Class: Handles inventory management tasks, including GUI creation, inventory display, revenue calculation, and order searching.
  • TimesheetSystem Class: Manages employee timesheets, including GUI creation, timeslot assignment, and employee addition/removal.
  • Main Function: Initializes the Tkinter application and starts the Pizza Ordering System.

File Structure

  • orders.json: A JSON file used for storing order data persistently.
  • Group Project 3: A directory containing pizza images and topping images used in the GUI.

Usage

Run the Python script (TigersPizza.py).

The main GUI window will appear, allowing customers to place orders or administrators to manage inventory and employee timesheets.

Customers can select their desired pizza, customize it, and enter their contact information to place an order.

Administrators can view the current inventory, track revenue, and manage employee timesheets.

The system provides error messages for invalid inputs and ensures inventory sufficiency before order placement.

Conclusion

The Pizza Ordering and Inventory Management System provides an efficient solution for managing pizza orders, inventory, and employee schedules in a restaurant setting. With its user-friendly interface and robust functionalities, it streamlines the ordering process for customers and simplifies inventory and employee management tasks for administrators.
